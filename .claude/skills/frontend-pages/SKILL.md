## Instructions

1. **Layout structure**
   - Use semantic HTML (`<header>`, `<main>`, `<section>`, `<footer>`)
   - Create responsive grids or flex layouts
   - Maintain consistent spacing and alignment

2. **Components**
   - Build reusable UI components (buttons, cards, forms, modals)
   - Use classes or props for customization
   - Ensure accessibility (ARIA labels, keyboard navigation)

3. **Styling**
   - Apply CSS, SCSS, or Tailwind for consistent design
   - Use CSS variables or theme files for colors and typography
   - Add hover, focus, and active states for interactivity

4. **Animations & Effects**
   - Smooth transitions for hover and state changes
   - Fade-in or slide-in elements
   - Optional parallax or scroll-based animations

## Best Practices
- Keep components small and reusable
- Follow consistent naming conventions
- Optimize images and assets
- Use mobile-first design
- Ensure text readability and color contrast
- Keep layouts simple and intuitive
